# QA Framework

Initial planned QA categories:

- Duplicate IDs
- Missing required attributes
- Invalid status values
- Invalid geometry
- Disconnected assets
- Assets outside expected service area
- Asset proximity warnings
- Overlapping features
- Tiny line segments
- Missing project references
- Coordinate-system problems
- CAD layers without mappings

The current repository includes placeholder checks only. Real QA rules must be configured against approved source schemas.
